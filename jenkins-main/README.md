# Secure Jenkins & Traefik CI/CD

This infrastructure sets up a hardened Jenkins instance behind a Traefik reverse proxy. It enables Jenkins to build and push Docker images to the **GitLab Container Registry** by securely bridging the host's Docker socket without using root privileges.

## 🏗 Prerequisites

- A Linux Host (Ubuntu/Debian recommended).
- GitLab Access Token (Scopes: `read_repository`, `write_registry`).

## Getting started

### 1. Clone this repository

```bash
git clone https://gitlab.csmju.com/kraken/jenkins.git
cd jenkins
```

### 2. Initialize the Host

Run your setup script on the host machine to install the Docker engine, create the `letsencrypt` storage, and set directory permissions.

```bash
chmod +x setup_docker_jenkins.sh
sudo ./setup_docker_jenkins.sh
```

### 2. Identify the Docker Group ID

```bash
getent group docker | cut -d: -f3
```

### 3. Configure the deployment

Open docker-compose.yml and update the group_add value under Jenkins service with the ID you found.

```bash
jenkins:
    container_name: jenkins
    image: jenkins/jenkins:lts-jdk21
    user: jenkins
    group_add:
      - "998"  # <--- REPLACE THIS WITH YOUR HOST'S DOCKER GID
    restart: unless-stopped
    volumes:
      - "jenkins_data:/var/jenkins_home"
      - "/var/run/docker.sock:/var/run/docker.sock"
    # ... (rest of your configuration)
```

### 4. Deploy

```bash
docker compose up -d
```

# Jenkins Configuration

Once Jenkins is deployed, complete these final steps to enable Docker builds and GitLab integration.

---

### 1. Install Required Plugins

You need the specific plugins that allow Jenkins to use Groovy DSL for Docker commands.

1.  Navigate to **Manage Jenkins** > **Plugins** > **Available Plugins**.
2.  Search for and install:
    - **Docker Pipeline**
    - **Docker**
3.  Restart Jenkins after the installation is complete.

---

### 2. Configure Docker CLI Tool

This allows Jenkins to automatically download the Docker binary so it can execute commands.

1.  Navigate to **Manage Jenkins** > **Tools**.
2.  Find the **Docker** section and click **Add Docker**.
3.  **Name**: `docker` (must be lowercase to match the `Jenkinsfile`).
4.  **Install automatically**: Enabled.
5.  **Add Installer**: Select **Download from docker.com**.
6.  Click **Save**.

---

### 3. Add GitLab Credentials

These credentials allow Jenkins to pull your private code and push the built images back to the GitLab Registry.

1.  Navigate to **Manage Jenkins** > **Credentials** > **System** > **Global credentials (unrestricted)**.
2.  Click **Add Credentials** and fill in the following:
    - **Kind**: Username with password
    - **Username**: Your GitLab username (or `deploy-token` if using one).
    - **Password**: Your GitLab Personal Access Token.
    - **ID**: `gitlab-registry-creds` (This **must** match the ID used in your `Jenkinsfile`).
    - **Description**: GitLab Registry Access.
3.  Click **Create**.

---

### 4. Verification Stage

To ensure everything is connected, run your pipeline. If you see an error like `Permission Denied` on the socket, verify that your host's Docker GID matches the `group_add` value in your `docker-compose.yml`.
